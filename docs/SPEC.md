# ForgeMap Specification

## Overview

**ForgeMap** is a lightweight, MIT-licensed, source-generator-based object transformation library for .NET. It generates type conversion code at compile time, providing zero-reflection runtime execution with compile-time type safety.

### Goals

1. **MIT License** - Fully open source, no commercial restrictions
2. **Source Generator** - Compile-time code generation, no runtime reflection
3. **Lightweight** - Minimal API surface, focused on common transformation patterns
4. **High Performance** - Generated code is as fast as hand-written code
5. **Debuggable** - Generated code is readable and debuggable
6. **Easy Migration** - Familiar patterns for AutoMapper users

### Non-Goals

1. Not a drop-in replacement for AutoMapper (similar patterns, different API)
2. No runtime/dynamic mapping (compile-time only)
3. No support for mapping configurations that require runtime type discovery (polymorphic dispatch via `[ForgeAllDerived]` uses compile-time-known type sets)

### Package Structure

Users reference **one NuGet package** that bundles everything:

| Package | Target Framework | Contents |
|---------|------------------|----------|
| `ForgeMap` | netstandard2.0 | Attributes, interfaces, and bundled source generator |

The generator is packaged as an analyzer (`.dll` in `analyzers/dotnet/cs/`) inside the same NuGet package, following the standard pattern for source-generator NuGet distribution.

## API Specification

### 1. Forger Declaration

Users declare a partial class with the `[ForgeMap]` attribute. The source generator implements the partial methods.

```csharp
using ForgeMap;

[ForgeMap]
public partial class AppForger
{
    // Simple forging - properties matched by name
    public partial OrderDto Forge(OrderEntity source);

    // Multiple forging methods in one forger
    public partial UserDto Forge(UserEntity source);
    public partial ProductDto Forge(ProductEntity source);
}
```

**Generated Code:**

```csharp
// Auto-generated by ForgeMap
public partial class AppForger
{
    public partial OrderDto Forge(OrderEntity source)
    {
        if (source == null) return null;

        return new OrderDto
        {
            Id = source.Id,
            Name = source.Name,
            CreatedAt = source.CreatedAt,
            // ... all matching properties
        };
    }
}
```

### 2. Attributes Reference

#### 2.1 `[ForgeMap]` - Class Level

Marks a partial class as a forger. The generator will implement all partial methods.

```csharp
[ForgeMap]
public partial class AppForger { }
```

**Options:**

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `NullHandling` | `NullHandling` | `ReturnNull` | Behavior when source is null |
| `PropertyMatching` | `PropertyMatching` | `ByName` | How to match properties (`ByName` is case-sensitive) |
| `SuppressDiagnostics` | `string[]?` | `null` | Diagnostic IDs to suppress for this forger |

```csharp
[ForgeMap(NullHandling = NullHandling.ThrowException)]
public partial class StrictForger { }
```

#### 2.2 `[Ignore]` - Method Level

Excludes destination properties from forging. Applied to the forging method, not to parameters or properties.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Ignore single property
    [Ignore(nameof(OrderDto.InternalId))]
    public partial OrderDto Forge(OrderEntity source);

    // Ignore multiple properties
    [Ignore(nameof(UserDto.PasswordHash), nameof(UserDto.SecurityStamp))]
    public partial UserDto Forge(UserEntity source);
}
```

#### 2.3 `[ForgeProperty]` - Method Level

Forges a source property to a differently-named destination property.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Forge Title -> Name
    [ForgeProperty(nameof(ProductEntity.Title), nameof(ProductDto.Name))]
    public partial ProductDto Forge(ProductEntity source);

    // Multiple property forgings
    [ForgeProperty(nameof(OrderEntity.OrderId), nameof(OrderDto.Id))]
    [ForgeProperty(nameof(OrderEntity.PlacedAt), nameof(OrderDto.OrderDate))]
    public partial OrderDto Forge(OrderEntity source);
}
```

#### 2.4 `[ForgeFrom]` - Method Level

Forges a destination property using a custom expression or method.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Using a static method
    [ForgeFrom(nameof(OrderDto.TotalWithTax), nameof(CalculateTotalWithTax))]
    public partial OrderDto Forge(OrderEntity source);

    // The resolver method (in same class)
    private static decimal CalculateTotalWithTax(OrderEntity source)
        => source.Subtotal * (1 + source.TaxRate);
}
```

**Resolver Method Signatures Supported:**

```csharp
// Option 1: Takes source, returns value
private static TValue MethodName(TSource source);

// Option 2: Takes source property value, returns value
private static TValue MethodName(TSourceProperty value);
```

**Resolution Rules:**

1. `Method(TSource)` always receives the full source object.
2. `Method(TSourceProperty)` is valid only when the generator can infer a single source property for the destination member by convention or an accompanying `[ForgeProperty]`.
3. If both overloads exist, the exact `TSourceProperty` overload is preferred for that destination member.
4. If no unique source property can be inferred for the `TSourceProperty` overload, the generator reports `FM0009`.

**Example using the source-property overload:**

```csharp
[ForgeProperty(nameof(OrderEntity.CustomerName), nameof(OrderDto.Name))]
[ForgeFrom(nameof(OrderDto.Name), nameof(NormalizeName))]
public partial OrderDto Forge(OrderEntity source);

private static string NormalizeName(string value) => value.Trim();
```

In this case, the generator passes `source.CustomerName` to `NormalizeName`.

#### 2.5 `[ForgeWith]` - Method Level

Uses another forging method for nested objects or type conversion.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Nested object forging - uses Forge(AddressEntity) for Address property
    [ForgeWith(nameof(UserDto.Address), nameof(Forge))]
    public partial UserDto Forge(UserEntity source);

    public partial AddressDto Forge(AddressEntity source);
}
```

**Resolution Rules:**
1. The generator looks for a `Forge` method on the same class with matching source type
2. If not found, a compile error `FM0008` is raised
3. Nested forging applies recursively (nested objects can also have nested objects)
4. Null source properties produce null destination properties (no nested call invoked)

#### 2.6 `[UseExistingValue]` - Parameter Level

Forges into an existing destination instance instead of creating a new one.

```csharp
[ForgeMap]
public partial class AppForger
{
    public partial void ForgeInto(OrderEntity source, [UseExistingValue] OrderDto destination);
}
```

**ForgeInto Behavioral Contract:**

| Aspect | Behavior |
|--------|----------|
| **Return type** | Must be `void`; the destination is mutated in place |
| **Null source** | No-op (destination unchanged) when `NullHandling.ReturnNull`; throws when `NullHandling.ThrowException` |
| **Null destination** | Throws `ArgumentNullException` always (cannot forge into null) |
| **Property overwrite** | All mapped properties are overwritten unconditionally |
| **Unmapped dest properties** | Preserved (not cleared or modified) |
| **Collections** | Destination collection is cleared then repopulated, not merged |

**Generated Code:**

```csharp
public partial void ForgeInto(OrderEntity source, OrderDto destination)
{
    ArgumentNullException.ThrowIfNull(destination);
    if (source == null) return;  // Or throw based on NullHandling

    destination.Id = source.Id;
    destination.Name = source.Name;
    // ... all mapped properties overwritten
}
```

#### 2.7 `[ReverseForge]` - Method Level

Generates a reverse forging method automatically.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Generates both Forge(A -> B) and Forge(B -> A)
    [ReverseForge]
    public partial ProductDto Forge(ProductEntity source);

    // Reverse method signature (auto-generated)
    // public partial ProductEntity Forge(ProductDto source);
}
```

**Behavioral Contract:**

| Aspect | Behavior |
|--------|----------|
| **Constructor requirement** | Destination type must have a parameterless constructor OR all required constructor parameters must match source properties by name |
| **Property renaming** | `[ForgeProperty(A, B)]` becomes `[ForgeProperty(B, A)]` in reverse |
| **[Ignore]** | Ignored properties remain ignored in both directions |
| **[ForgeFrom]** | NOT reversed automatically (compile warning `FM0012` if no manual reverse provided) |
| **[ForgeWith]** | Reversed if the nested forging method also has `[ReverseForge]`; otherwise compile warning `FM0015` |
| **Ambiguous reverse** | If both forward and reverse methods are explicitly declared, the explicit one takes precedence |

#### 2.8 `[AfterForge]` - Method Level

Calls a method after forging completes for post-processing.

```csharp
[ForgeMap]
public partial class AppForger
{
    [AfterForge(nameof(EnrichOrder))]
    public partial OrderDto Forge(OrderEntity source);

    private static void EnrichOrder(OrderEntity source, OrderDto destination)
    {
        destination.DisplayName = $"Order #{source.Id} - {source.CustomerName}";
    }
}
```

#### 2.9 `[BeforeForge]` - Method Level

Calls a method before forging for pre-processing or validation.

```csharp
[ForgeMap]
public partial class AppForger
{
    [BeforeForge(nameof(ValidateSource))]
    public partial OrderDto Forge(OrderEntity source);

    private static void ValidateSource(OrderEntity source)
    {
        ArgumentNullException.ThrowIfNull(source.Id);
    }
}
```

**Execution Order:**

1. **Null check** (based on `NullHandling`) - if source is null, return null/throw immediately
2. **`[BeforeForge]`** callbacks execute (in declaration order)
3. **Property forging** occurs
4. **`[AfterForge]`** callbacks execute (in declaration order)
5. **Return** destination

**Note:** Since the null check runs first, `[BeforeForge]` callbacks are guaranteed to receive a non-null source when `NullHandling.ReturnNull` is set. With `NullHandling.ThrowException`, an exception is thrown before callbacks execute.

#### 2.10 `[ConvertWith]` - Method Level *(Future)*

> **Note:** This feature is planned for a future version. For v1.0, use `[ForgeFrom]` with a static method that contains the conversion logic.

Uses a custom converter class for complex type transformations. This is the ForgeMap equivalent of AutoMapper's `ITypeConverter` and `ConvertUsing()`.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Use a converter class for complex transformations
    [ConvertWith(typeof(OrderToInvoiceConverter))]
    public partial Invoice Forge(Order source);
}

// Converter class - must implement ITypeConverter<TSource, TDest>
public class OrderToInvoiceConverter : ITypeConverter<Order, Invoice>
{
    public Invoice Convert(Order source)
    {
        return new Invoice
        {
            InvoiceNumber = $"INV-{source.Id}",
            LineItems = source.Items.Select(x => new InvoiceLine
            {
                Description = x.ProductName,
                Amount = x.Price * x.Quantity
            }).ToList(),
            Total = source.Items.Sum(x => x.Price * x.Quantity)
        };
    }
}
```

Generated code checks `source` for null before invoking the converter, so converter implementations can assume non-null input when called through generated ForgeMap code.

**Lifecycle and Instantiation:**

| Aspect | Behavior |
|--------|----------|
| **Instantiation** | Converters are instantiated via `new T()` at each call site by default |
| **DI support** | If the forger class has a constructor accepting `IServiceProvider`, converters are resolved from DI; if it accepts `IServiceScopeFactory`, generated code creates a scope per invocation before resolving the converter |
| **Scoped converters** | Prefer registering the forger as `Scoped`, or inject `IServiceScopeFactory` so generated code resolves scoped converter dependencies from a fresh scope per invocation |
| **Stateless converters** | Recommended pattern; converter instances may be reused in future versions |
| **Null handling** | Generated code returns null before invoking the converter when source is null |

**DI Example:**

```csharp
[ForgeMap]
public partial class AppForger
{
    private readonly IServiceProvider _services;

    public AppForger(IServiceProvider services) => _services = services;

    // Converter resolved from DI: _services.GetRequiredService<MyConverter>()
    [ConvertWith(typeof(MyConverter))]
    public partial DestType Forge(SourceType source);
}
```

> **Warning:** If the forger is registered as **Singleton** but converters are **Scoped**, resolving them from the singleton's captured `IServiceProvider` uses the root provider instead of a request scope. In practice this typically fails under scope validation or promotes scoped dependencies to effectively root-scoped behavior. Inject `IServiceScopeFactory` instead, or register the forger as Scoped.

**Scoped DI Example:**

```csharp
[ForgeMap]
public partial class AppForger
{
    private readonly IServiceScopeFactory _scopeFactory;

    public AppForger(IServiceScopeFactory scopeFactory) => _scopeFactory = scopeFactory;

    [ConvertWith(typeof(MyScopedConverter))]
    public partial DestType Forge(SourceType source);
}

// Generated code when forger has IServiceScopeFactory constructor
public partial DestType Forge(SourceType source)
{
    if (source == null) return null;
    using var scope = _scopeFactory.CreateScope();
    var converter = scope.ServiceProvider.GetRequiredService<MyScopedConverter>();
    return converter.Convert(source);
}
```

**When to use `[ConvertWith]` vs `[ForgeFrom]`:**

| Scenario | Use |
|----------|-----|
| Simple property transformation | `[ForgeFrom]` with inline method |
| Complex multi-step conversion | `[ConvertWith]` with converter class |
| Reusable conversion logic | `[ConvertWith]` - converter can be shared |
| One-to-many property expansion | `[ConvertWith]` |
| Type requires constructor logic | `[ConvertWith]` |

### 3. Collection Forging

Collections are forged automatically when element types have a corresponding forging method.

```csharp
[ForgeMap]
public partial class AppForger
{
    // Element forging
    public partial ProductDto Forge(ProductEntity source);

    // Collection forgings - auto-generated based on element forging
    public partial List<ProductDto> Forge(List<ProductEntity> source);
    public partial ProductDto[] Forge(ProductEntity[] source);
    public partial IEnumerable<ProductDto> Forge(IEnumerable<ProductEntity> source);
}
```

**Resolution Rules:**

1. **Auto-detection:** If a `Forge(TElement source)` method exists, collection methods are auto-generated
2. **Explicit override:** Declaring an explicit collection method overrides auto-generation
3. **Null source:** Returns null (not empty collection) when source collection is null
4. **Empty source:** Returns empty collection of target type
5. **Element null:** Null elements in source produce null elements in destination

**Supported Collection Types:**
- `T[]` (arrays)
- `List<T>`
- `IList<T>` (returns `List<T>`)
- `ICollection<T>` (returns `List<T>`)
- `IEnumerable<T>` (returns lazy `Select()` projection)
- `IReadOnlyList<T>` (returns `List<T>`)
- `IReadOnlyCollection<T>` (returns `List<T>`)

**Generated Code Example:**

```csharp
// For List<T>
public partial List<ProductDto> Forge(List<ProductEntity> source)
{
    if (source == null) return null;
    var result = new List<ProductDto>(source.Count);
    foreach (var item in source)
    {
        result.Add(Forge(item));
    }
    return result;
}

// For IEnumerable<T> - lazy evaluation
public partial IEnumerable<ProductDto> Forge(IEnumerable<ProductEntity> source)
{
    if (source == null) return null;
    return source.Select(item => Forge(item));
}
```

### 4. Enum Forging

#### 4.1 Enum to Enum (by name or value)

```csharp
public enum OrderStatus { Pending, Processing, Shipped, Delivered }
public enum OrderStatusDto { Pending, Processing, Shipped, Delivered }

[ForgeMap]
public partial class AppForger
{
    // Enums with matching names forge automatically
    public partial OrderStatusDto Forge(OrderStatus source);
}
```

#### 4.2 Enum to String

```csharp
[ForgeMap]
public partial class AppForger
{
    // Enum -> string uses .ToString()
    public partial string Forge(OrderStatus source);
}
```

#### 4.3 String to Enum

```csharp
[ForgeMap]
public partial class AppForger
{
    // string -> Enum uses Enum.Parse with case-insensitive matching
    public partial OrderStatus Forge(string source);
}
```

#### 4.4 Custom Enum Conversion

For complex enum conversions, use `[ForgeFrom]`:

```csharp
[ForgeMap]
public partial class AppForger
{
    [ForgeFrom(nameof(OrderDto.StatusText), nameof(ConvertStatus))]
    public partial OrderDto Forge(OrderEntity source);

    private static string ConvertStatus(OrderStatus status) => status switch
    {
        OrderStatus.Pending => "Awaiting Processing",
        OrderStatus.Processing => "In Progress",
        OrderStatus.Shipped => "On the Way",
        OrderStatus.Delivered => "Complete",
        _ => "Unknown"
    };
}
```

### 5. Nullable Handling

**Null Handling Contract:**

| Scenario | Default Behavior | Override |
|----------|-----------------|----------|
| Source object is null | Return null | `[ForgeMap(NullHandling = NullHandling.ThrowException)]` |
| Source property is null (nullable ref → non-nullable ref) | Null-forgiving (`!`) | `[ForgeMap(NullPropertyHandling = ...)]` or per-property via `[ForgeProperty]` ([v1.2 spec](SPEC-v1.2-null-property-handling.md)) |
| Nullable<T> → T | Use `.Value` (throws if null) | Use `[ForgeFrom]` for safe default |
| T → Nullable<T> | Direct assignment | N/A |
| Collection is null | Return null | N/A |
| Collection element is null | Preserve null in output | N/A |

#### 5.1 Nullable Value Types

```csharp
public class OrderEntity { public DateTime? ShippedAt { get; set; } }
public class OrderDto { public DateTime ShippedAt { get; set; } }

[ForgeMap]
public partial class AppForger
{
    // DateTime? -> DateTime unwraps via .Value (throws at runtime if null)
    // Or provide default via [ForgeFrom]
    [ForgeFrom(nameof(OrderDto.ShippedAt), nameof(GetShippedAt))]
    public partial OrderDto Forge(OrderEntity source);

    private static DateTime GetShippedAt(OrderEntity s)
        => s.ShippedAt ?? DateTime.MinValue;
}
```

#### 5.2 Nullable Reference Types

With nullable reference types enabled, the generator respects nullability annotations and applies the configured `NullPropertyHandling` strategy (see [v1.2 spec](SPEC-v1.2-null-property-handling.md)):

```csharp
public class Source { public string? Name { get; set; } }
public class Dest { public string Name { get; set; } }  // Non-nullable

// Generator applies NullPropertyHandling strategy (default: NullForgiving)
// Generated: target.Name = source.Name!;
// Also reports FM0007 warning: The nullable source 'Source.Name' is mapped to non-nullable destination 'Dest.Name'
```

### 6. Constructor Mapping

When destination has no parameterless constructor, the generator uses constructor parameters:

```csharp
public record OrderDto(string Id, string CustomerName, DateTime OrderDate);

[ForgeMap]
public partial class AppForger
{
    // Generator uses constructor - matches parameters by name
    public partial OrderDto Forge(OrderEntity source);
}
```

**Constructor Selection Rules:**

1. **Parameterless first:** If available, use parameterless constructor + property setters
2. **Primary constructor:** For records, use the primary constructor
3. **Best match:** Select constructor where most parameters match source properties by name (constructor parameter matching is always case-insensitive, independent of `PropertyMatching` setting)
4. **Ambiguity:** If multiple constructors match equally, compile error `FM0013`
5. **Unmapped params:** Constructor parameters without matching source properties cause error `FM0014`

**`[ForgeProperty]` with Constructors:**

When `[ForgeProperty]` remaps a source property, the constructor parameter matching uses the **destination property name** (not source):

```csharp
public record UserDto(string FullName);  // Constructor param: FullName

[ForgeProperty(nameof(UserEntity.Name), nameof(UserDto.FullName))]
public partial UserDto Forge(UserEntity source);

// Generated: new UserDto(FullName: source.Name)
```

The generator maps `source.Name` → constructor parameter `FullName` because `[ForgeProperty]` declared that `Name` maps to `FullName`.

**Generated:**

```csharp
public partial OrderDto Forge(OrderEntity source)
{
    if (source == null) return null;

    return new OrderDto(
        Id: source.Id,
        CustomerName: source.CustomerName,
        OrderDate: source.OrderDate
    );
}
```

**Hybrid Pattern (constructor + setters):**

```csharp
public class Entity
{
    public Entity(string id) { Id = id; }  // Required
    public string Id { get; }
    public string Name { get; set; }  // Optional, set after construction
}

// Generated code:
// var result = new Entity(source.Id);
// result.Name = source.Name;
// return result;
```

### 7. Flattening and Unflattening

#### 7.1 Automatic Flattening

```csharp
public class OrderEntity
{
    public CustomerInfo Customer { get; set; }
}
public class CustomerInfo
{
    public string Name { get; set; }
}
public class OrderDto
{
    public string CustomerName { get; set; }  // Flattened
}

// Generator auto-flattens: source.Customer.Name -> dest.CustomerName
```

#### 7.2 Explicit Nested Path

Use dot-notation string literals (not `nameof()`) for nested paths, since `nameof()` cannot express paths:

```csharp
// Correct: string literal for nested path
[ForgeProperty("Customer.Email", nameof(OrderDto.ContactEmail))]
public partial OrderDto Forge(OrderEntity source);

// Also valid: source property via nameof when not nested
[ForgeProperty(nameof(OrderEntity.Title), nameof(OrderDto.Name))]
public partial OrderDto Forge(OrderEntity source);
```

**Path Resolution:**
- Paths are split by `.` and resolved property-by-property
- Each segment must be a valid public property
- Null checks are inserted for each intermediate segment

## Diagnostics and Errors

The generator produces compile-time diagnostics for common issues:

| Code | Severity | Description |
|------|----------|-------------|
| `FM0001` | Error | Forger class must be partial |
| `FM0002` | Error | Forging method must be partial |
| `FM0003` | Error | Source type has no accessible properties |
| `FM0004` | Error | Destination type has no accessible constructor |
| `FM0005` | Warning | Unmapped source property |
| `FM0006` | Warning | Unmapped destination property |
| `FM0007` | Warning | Nullable source mapped to non-nullable destination |
| `FM0008` | Error | Referenced resolver method not found |
| `FM0009` | Error | Resolver method has invalid signature |
| `FM0010` | Error | Circular mapping dependency detected |
| `FM0011` | Info | Property mapped by convention |
| `FM0012` | Warning | `[ForgeFrom]` cannot be auto-reversed |
| `FM0013` | Error | Ambiguous constructor selection |
| `FM0014` | Error | Constructor parameter has no matching source |
| `FM0015` | Warning | `[ForgeWith]` nested method lacks `[ReverseForge]` |
| `FM0016` | Error | Hook method not found or has invalid signature |
| `FM0017` | Error | `[UseExistingValue]` on non-reference type or method returns non-void |
| `FM0018` | Warning | `[BeforeForge]`/`[AfterForge]` not supported on enum or collection forge methods |
| `FM0019` | Error | `[IncludeBaseForge]` base forge method not found |
| `FM0020` | Error | `[IncludeBaseForge]` type mismatch: not an inheritance relationship |
| `FM0021` | Info | `[IncludeBaseForge]` inherited attribute overridden by explicit attribute |
| `FM0022` | Warning | `[ForgeAllDerived]` found no derived forge methods |
| `FM0023` | Error | `[ForgeAllDerived]` cannot be combined with `[ConvertWith]` |

### Suppressing Warnings

```csharp
// Suppress specific unmapped property warning
[Ignore(nameof(Dest.UnusedProperty))]
public partial Dest Forge(Source source);

// Or at forger level
[ForgeMap(SuppressDiagnostics = new[] { "FM0005", "FM0006" })]
public partial class AppForger { }
```

## Dependency Injection Integration

### ASP.NET Core Registration

```csharp
// In Program.cs or Startup.cs
services.AddSingleton<AppForger>();

// Or with factory for DI-aware converters
services.AddSingleton<AppForger>(sp => new AppForger(sp));

// Or scan assembly for all forgers
services.AddForgeMaps(typeof(Program).Assembly);
```

**`AddForgeMaps` Specification:**

```csharp
public static class ForgeMapServiceCollectionExtensions
{
    /// <summary>
    /// Registers all classes marked with [ForgeMap] in the specified assembly.
    /// </summary>
    /// <param name="services">The service collection.</param>
    /// <param name="assembly">Assembly to scan for forger classes.</param>
    /// <param name="lifetime">Service lifetime (default: Singleton).</param>
    public static IServiceCollection AddForgeMaps(
        this IServiceCollection services,
        Assembly assembly,
        ServiceLifetime lifetime = ServiceLifetime.Singleton);
}
```

**Behavior:**
- Scans for all types with `[ForgeMap]` attribute
- Registers each as itself (e.g., `services.AddSingleton<AppForger>()`)
- Uses parameterless constructor if available; otherwise factory with `IServiceProvider`
- Skips abstract classes and interfaces
- The default Singleton lifetime is intended for stateless forgers or singleton/transient converter dependencies; use `ServiceLifetime.Scoped` when converter dependencies are scoped

### Usage with DI

```csharp
public class OrderService
{
    private readonly AppForger _forger;

    public OrderService(AppForger forger)
    {
        _forger = forger;
    }

    public OrderDto GetOrder(string id)
    {
        var entity = _repository.Get(id);
        return _forger.Forge(entity);
    }
}
```

### Converter DI Resolution

When forgers are constructed with `IServiceProvider`, converters used in `[ConvertWith]` are resolved from DI:

```csharp
// Generated code when forger has IServiceProvider constructor
public partial DestType Forge(SourceType source)
{
    if (source == null) return null;
    var converter = _services.GetRequiredService<MyConverter>();
    return converter.Convert(source);
}
```

## Migration Guide from AutoMapper

### Pattern Mapping

| AutoMapper | ForgeMap |
|------------|-----------|
| `CreateMap<S, D>()` | `public partial D Forge(S source);` |
| `.ForMember(d => d.X, o => o.Ignore())` | `[Ignore(nameof(D.X))]` |
| `.ForMember(d => d.X, o => o.MapFrom(s => s.Y))` | `[ForgeProperty(nameof(S.Y), nameof(D.X))]` |
| `.ForMember(d => d.X, o => o.MapFrom(s => Calc(s)))` | `[ForgeFrom(nameof(D.X), nameof(Calc))]` |
| `.ConvertUsing<TConverter>()` | `[ConvertWith(typeof(TConverter))]` *(future)* |
| `.ConvertUsing(new Converter())` | `[ConvertWith(typeof(Converter))]` *(future)* |
| `ITypeConverter<S, D>` | `ITypeConverter<S, D>` *(future)* |
| `.IncludeBase<TBaseSrc, TBaseDst>()` | `[IncludeBaseForge(typeof(TBaseSrc), typeof(TBaseDst))]` |
| `.Include<TDerivedSrc, TDerivedDst>()` | Auto-discovered by `[ForgeAllDerived]` |
| `.IncludeAllDerived()` | `[ForgeAllDerived]` |
| `.ReverseMap()` | `[ReverseForge]` |
| `.AfterMap((s, d) => { ... })` | `[AfterForge(nameof(Method))]` |
| `.BeforeMap((s, d) => { ... })` | `[BeforeForge(nameof(Method))]` |
| `mapper.Map<D>(source)` | `forger.Forge(source)` |
| `mapper.Map(source, existing)` | `forger.ForgeInto(source, existing)` |

**Notes:**

- Simple property-access `MapFrom(s => s.Y)` maps to `[ForgeProperty]`.
- Computed `MapFrom` lambdas map to `[ForgeFrom]`.
- Converter-based AutoMapper features are planned for a future version.
- Inheritance/polymorphism features (`[IncludeBaseForge]`, `[ForgeAllDerived]`) are available since v1.1.

### Example Migration

**Before (AutoMapper):**

```csharp
public class MappingProfile : Profile
{
    public MappingProfile()
    {
        CreateMap<OrderEntity, OrderDto>()
            .ForMember(d => d.Id, o => o.MapFrom(s => s.OrderId))
            .ForMember(d => d.OrderDate, o => o.MapFrom(s => s.PlacedAt))
            .ForMember(d => d.TotalWithTax, o => o.MapFrom(s => s.Subtotal * (1 + s.TaxRate)))
            .ForMember(d => d.InternalNotes, o => o.Ignore());
    }
}

// Usage
var result = _mapper.Map<OrderDto>(entity);
```

**After (ForgeMap):**

```csharp
[ForgeMap]
public partial class AppForger
{
    [ForgeProperty(nameof(OrderEntity.OrderId), nameof(OrderDto.Id))]
    [ForgeProperty(nameof(OrderEntity.PlacedAt), nameof(OrderDto.OrderDate))]
    [ForgeFrom(nameof(OrderDto.TotalWithTax), nameof(CalculateTotalWithTax))]
    [Ignore(nameof(OrderDto.InternalNotes))]
    public partial OrderDto Forge(OrderEntity source);

    private static decimal CalculateTotalWithTax(OrderEntity s)
        => s.Subtotal * (1 + s.TaxRate);
}

// Usage
var result = _forger.Forge(entity);
```

## Configuration Options

### Global Configuration (Optional)

For project-wide defaults, use the `[ForgeMapDefaults]` assembly attribute:

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `NullHandling` | `NullHandling` | `ReturnNull` | Default null handling mode for all forgers in the assembly |
| `GenerateCollectionMappings` | `bool` | `true` | Enables auto-generation of collection mapping methods when element mappings exist |
| `PropertyMatching` | `PropertyMatching` | `ByName` | Default property matching mode for all forgers in the assembly |
| `NullPropertyHandling` | `NullPropertyHandling` | `NullForgiving` | Default strategy for nullable-to-non-nullable ref property assignments ([v1.2 spec](SPEC-v1.2-null-property-handling.md)) |

```csharp
// In AssemblyInfo.cs or any file
[assembly: ForgeMapDefaults(
    NullHandling = NullHandling.ReturnNull,
    GenerateCollectionMappings = true,
    PropertyMatching = PropertyMatching.ByNameCaseInsensitive,
    NullPropertyHandling = NullPropertyHandling.CoalesceToDefault
)]
```

## Limitations (v1.2)

The following AutoMapper features are **not supported** in v1.2:

| Feature | Reason | Workaround |
|---------|--------|------------|
| Runtime type mapping | Requires reflection | Use explicit generic methods |
| Open generics | Complex source generation | Define concrete mappings |
| `IValueResolver` with DI | Compile-time limitation | Use `[ForgeFrom]` with static method; `[ConvertWith]` in a future version |
| Conditional mapping | Complex to generate | Use `[ForgeFrom]` with logic |
| `ProjectTo<T>()` for EF | Requires expression trees | Future version |
| Inline configuration | Compile-time only | Use attributes |

## Performance Characteristics

### Projected Benchmarks

**Note:** These are projected estimates based on source-generator architecture analysis. Actual benchmarks will be published after v1.0 implementation.

| Scenario | AutoMapper (est.) | ForgeMap (proj.) | Projected Gain |
|----------|-------------------|-------------------|----------------|
| Simple object (10 props) | ~150ns | ~15-30ns | 5-10x |
| Nested object (3 levels) | ~450ns | ~45-90ns | 5-10x |
| Collection (100 items) | ~15μs | ~1.5-3μs | 5-10x |
| Large object (50 props) | ~750ns | ~75-150ns | 5-10x |

*Performance claims will be validated with BenchmarkDotNet in the v1.0 release.*

### Why ForgeMap is Faster

1. **No reflection** - All mapping code generated at compile time
2. **No expression compilation** - No runtime JIT for mapping expressions
3. **No dictionary lookups** - No type map resolution at runtime
4. **Inlined code** - Generated code can be further optimized by JIT

## Appendix B: Extensibility Guidelines

To ensure ForgeMap can easily add new features, follow these design principles:

### 1. Attribute-Based Extension

All forging behaviors are controlled by attributes. New features should:
- Add new attributes rather than modifying existing ones
- Keep attributes composable (multiple attributes on same method)
- Use compile-time validation for attribute parameters

### 2. Generator Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│  Roslyn Incremental Source Generator                            │
├─────────────────────────────────────────────────────────────────┤
│  1. Syntax Provider (find [ForgeMap] classes)                 │
│  2. Semantic Analysis (resolve types, validate)                 │
│  3. Mapping Plan Builder (determine property forgings)          │
│  4. Code Emitter (generate C# code)                            │
│  5. Diagnostic Reporter (errors, warnings)                      │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Future Feature Hooks

| Future Feature | Extension Point |
|----------------|-----------------|
| ProjectTo support | Add `IQueryable` return type detection |
| Conditional mapping | Add `[ForgeWhen]` attribute |
| Custom naming conventions | Add `PropertyMatching.Custom` with delegate |
| Async converters | Add `IAsyncTypeConverter<S, D>` interface |

### 4. Testing Strategy

- Unit tests for each attribute behavior
- Integration tests with real type mappings
- Snapshot tests for generated code verification
- Performance benchmarks comparing to AutoMapper
